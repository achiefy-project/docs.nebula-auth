# Сохранение паролей

NebulaAuth позволяет сохранять пароли аккаунтов, чтобы автоматически восстанавливать сессию без вашего участия. Также эта функция позволяет быстро копировать пароль с помощью контекстного меню аккаунта (см. [Контекстное меню](../../getting-started/interface.md#контекстное-меню-аккаунта)).

***

### Как это работает

При необходимости (например, если сессия истекла) приложение может выполнить повторный вход в аккаунт.

Для этого используется сохранённый пароль.

Если включено сохранение пароля:

* он записывается в **maFile**
* используется для автоматического входа
* не требует ручного ввода при сбое сессии

***

### Защита паролей

Пароли не хранятся в открытом виде.

Для их защиты используется **пароль шифрования**, который задаётся в NebulaAuth. (см. [Настройки безопасности](../settings/main.md#Пароль-шифрования))

* пароль аккаунта сохраняется в зашифрованном виде
* пароль шифрования не хранится на диске
* после перезапуска его нужно ввести заново

Если вы забыли пароль шифрования, восстановить сохранённые пароли невозможно. Это не повлияет на работу аккаунтов, но автоматический вход работать не будет, и вам нужно будет заново назначить пароли для каждого аккаунта.

***

### Что важно понимать

* шифруется только пароль аккаунта, а не весь maFile
* без пароля шифрования автоматический вход работать не будет
* при переносе maFile зашифрованный пароль переносится вместе с ним

***

### Массовое назначение паролей

Вы можете записать пароли сразу для нескольких аккаунтов.

#### Формат

Каждая строка соответствует одному аккаунту:

```id="mass-pass"
login:password
SteamID:password
```

***

#### Как использовать

1. Откройте Меню → Другое → Назначить пароли
2. Введите пароль шифрования, если он ещё не задан
3. Вставьте список строк
4. Подтвердите применение

Формат должен быть одним из следующих:

* `login:password`
* `SteamID:password`

После применения вы увидите результат с количеством успешно назначенных паролей.

## ❓ Частые вопросы

<details>

<summary><strong>Как убрать запрос пароля при запуске?</strong></summary>

Очистите поле пароля шифрования в настройках и сохраните изменения.

После этого NebulaAuth не будет запрашивать пароль при запуске.

См. [Настройки → Пароль шифрования](../settings/main.md#пароль-шифрования)

</details>
