# Работа с сессией

Сессия — это состояние входа в аккаунт Steam.

Она хранится внутри **maFile** и позволяет выполнять действия без повторного ввода пароля.

***

### Как это работает

После входа NebulaAuth сохраняет мобильную сессию аккаунта.

Обычно она действует длительное время (в среднем до нескольких месяцев), поэтому вводить пароль каждый раз не требуется.

Однако сессия может сбрасываться:

* при смене IP-адреса
* при смене пароля
* по внутренним причинам Steam

***

### Если сессия слетела

Если появляется ошибка сессии, выполните повторный вход:

1. Выберите аккаунт
2. Нажмите **Аккаунт → Войти**
3. Введите пароль

После этого сессия будет обновлена и сохранена в maFile.

***

### Автоматический вход

Вы можете включить сохранение пароля.

В этом случае NebulaAuth сможет автоматически восстанавливать сессию без вашего участия.

Пароль сохраняется в зашифрованном виде.

Подробнее: [saving-password.md](saving-password.md "mention")

***

### После переноса из SDA

После импорта maFile из Steam Desktop Authenticator все аккаунты будут показывать ошибку **«Сессия полностью истекла»**.

Это нормально.

После переноса необходимо **один раз выполнить вход** для каждого аккаунта.

***

### Кнопка «Обновить сессию»

В Steam используются два типа сессий:

* длительная (основная)
* короткая (рабочая)

Кнопка **«Обновить сессию»** обновляет короткую сессию, если это возможно.

{% hint style="info" %}
В новых версиях NebulaAuth это обычно не требуется — приложение пытается восстановить сессию автоматически.
{% endhint %}
