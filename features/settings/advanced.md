# Дополнительные настройки (WIP)

В этом разделе параметры, которые напрямую влияют на прокси и автоподтверждения.

## Настройки отображения прокси

Параметры:

* `ProxyManagerDisplayProtocol`
* `ProxyManagerDisplayCredentials`

Они управляют только отображением в менеджере прокси и не меняют саму сетевую логику.

## Параметры автоподтверждений

### TimerSeconds

* основной интервал таймера в секундах
* минимальное значение в UI: 5 секунд

### MaacErrorThreshold

* через сколько времени непрерывных ошибок аккаунт переводится в статус ошибки
* по умолчанию: 3 часа

### MaacRetryInterval

* интервал повторной попытки после ошибки в warning-состоянии
* по умолчанию: 15 минут

## Как применяется логика ошибок

MAAC использует статусы:

* `Ok` — все работает
* `Warning` — ошибка есть, но будут повторы через `MaacRetryInterval`
* `Error` — ошибка держится дольше `MaacErrorThreshold`

Когда проблема исчезает, статус возвращается в `Ok`.

Выберите аккаунт в списке и посмотрите на индикатор автоподтверждений в правой панели:

* **Зелёный** — OK
* **Жёлтый** — Warning
* **Красный** — Error

**Влияют ли настройки MAAC на ручные подтверждения?**

Нет, настройки MAAC влияют только на **автоматические** подтверждения. Ручные подтверждения (через кнопку "Загрузить подтверждения") работают независимо.

**Можно ли настроить разные интервалы для разных аккаунтов?**

Нет, интервал таймера единый для всех аккаунтов. Однако вы можете запустить **несколько копий NebulaAuth** с разными настройками для разных групп аккаунтов.
