# Добавление аккаунтов

Данный раздел ознакомит вас с различными способами добавления аккаунтов в NebulaAuth. Подробнее о каждом способе вы можете узнать, перейдя по соответствующим ссылкам.

### 🧭 Какой способ выбрать?

Приложение поддерживает три основных способа добавления аккаунтов, каждый из которых подходит для разных ситуаций

<details>

<summary><strong>✅ Способ 1: Импорт готовых мафайлов</strong></summary>

Подходит, если у вас уже есть мафайлы (`.maFile`) — например, из Steam Desktop Authenticator или другого источника.

Вы можете импортировать мафайлы любым удобным способом:

* **Перетаскивание (Drag & Drop)** Просто перетащите `.maFile` файлы прямо в окно NebulaAuth.
* **Из буфера обмена**
  1. Выделите мафайлы в проводнике
  2. Нажмите **Ctrl + C**
  3. Перейдите в окно NebulaAuth
  4. Нажмите **Ctrl + V**
* **Через меню**
  1. Нажмите **Файл → Импорт**
  2. Выберите `.maFile` на компьютере
  3. Нажмите **Открыть**

Подробнее: [import.md](import.md "mention")

</details>

<details>

<summary><strong>🔗 Способ 2: Привязка нового аккаунта</strong></summary>

Подходит, если на аккаунте **ещё не подключён Steam Guard**.

1. Нажмите **Аккаунт → Привязать**
2. Следуйте инструкциям мастера привязки

После привязки действует стандартное ограничение Steam на обмены — **7-15 дней**.

Подробнее: [link-new.md](link-new.md "mention")

</details>

<details>

<summary><strong>📱 Способ 3: Перенос Steam Guard с телефона (2 дня трейдбан)</strong></summary>

Подходит, если Steam Guard уже активен на мобильном устройстве.

1. Нажмите **Аккаунт → Перенести Steam Guard**
2. Следуйте инструкциям мастера переноса

После переноса обычно действует **2 дня** задержки обменов.

Подробнее: [transfer.md](transfer.md "mention")

</details>

### 📁 Где хранятся мафайлы?

Все импортированные мафайлы сохраняются в папке `maFiles`, которая находится рядом с файлом **NebulaAuth.exe**.

Чтобы быстро открыть эту папку:

* нажмите **Файл → Открыть папку**

### 📄 Что происходит при добавлении аккаунта?

При любом способе добавления рядом с приложением создаётся папка `maFiles`. В ней хранится мафайл — файл с расширением `.maFile`.

Этот файл содержит **всю информацию** для:

* генерации Guard-кодов
* подтверждения обменов
* подтверждения входов

{% hint style="warning" %}
Мафайл — это самый важный файл для Steam Guard. Без него восстановить двухфакторную аутентификацию будет невозможно.
{% endhint %}

Рекомендуем:

* делать резервные копии папки `maFiles`
* никогда не удалять или модифицировать эти файлы вручную
* хранить R-Code в надёжном месте лучше всего — **на бумаге**, а не в цифровом виде

После добавления аккаунтов NebulaAuth готов к работе. Если ваш способ работы подразумевает использование прокси — не забудьте настроить их перед тем как начинать взаимодействовать с аккаунтами (_подробнее в разделе_ [proxy](../../features/proxy/ "mention")).

В следующих разделах вы можете ознакомиться с главными функциями приложения:

* [interface.md](../interface.md "mention") — основные элементы интерфейса и их назначение
* [first-steps.md](../first-steps.md "mention") — быстрый старт для получения кодов и подтверждения действий

### ❓ Частые вопросы

<details>

<summary><strong>Можно ли использовать один аккаунт в нескольких приложениях одновременно?</strong></summary>

Да, это возможно, если у вас есть **maFile**.

Файл можно использовать на нескольких устройствах или в разных приложениях одновременно — он содержит все необходимые данные.

Если же Steam Guard привязан к мобильному приложению, получить **maFile** напрямую невозможно без вмешательства в систему устройства.

</details>
