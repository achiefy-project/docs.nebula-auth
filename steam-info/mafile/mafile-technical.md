# Техническая информация о мафайле

{% hint style="info" %}
Эта статья предназначена для продвинутых пользователей.
{% endhint %}

***

### Базовые поля Steam Guard

maFile представляет собой JSON-файл, содержащий данные Steam Guard.

Основные поля:

* `shared_secret`\
  Используется для генерации кодов входа (Steam Guard)
* `identity_secret`\
  Используется для подтверждения действий (обмены, продажи)
* `revocation_code`\
  Код для отвязки Steam Guard от аккаунта
* `account_name`\
  Логин аккаунта Steam
* `steamid`\
  Уникальный идентификатор аккаунта
* `device_id`\
  Сгенерированный идентификатор устройства. Используется при работе с подтверждениями.

***

### Дополнительные поля Steam

В maFile могут присутствовать служебные поля Steam:

* `serial_number`
* `server_time`
* `token_gid`
* `secret_1`
* `uri`

Эти поля либо не используются в процессе работы, либо их назначение неизвестно.\
В NebulaAuth они не задействуются и не влияют на работу аккаунта.

***

### Сессия

В maFile может храниться сессия аккаунта (данные авторизации).

В NebulaAuth используется современный формат сессии, аналогичный официальному приложению Steam:

* `access_token` — используется для выполнения запросов
* `refresh_token` — используется для продления сессии без повторного входа

Если `access_token` устарел, NebulaAuth автоматически пытается обновить его через `refresh_token`.

Если это невозможно (например, refresh-токен истёк), потребуется повторный вход в аккаунт.

{% hint style="info" %}
В Steam Desktop Authenticator (SDA) используется устаревший формат — закодированный session-токен.

Из-за этого при импорте maFile из SDA сессия не может быть использована, и требуется один раз выполнить вход в аккаунт.
{% endhint %}

### Поля NebulaAuth

NebulaAuth добавляет собственные поля:

* `Proxy`\
  Прокси, назначенный аккаунту
* `Group`\
  Группа аккаунта
* `Password`\
  Сохранённый пароль (в зашифрованном виде)

Эти поля используются только внутри NebulaAuth и не являются частью формата Steam Guard.

### ❓ Частые вопросы

<details>

<summary><strong>Можно ли редактировать maFile вручную?</strong></summary>

Технически — да.

Но это не рекомендуется: ошибка в структуре файла может сделать его непригодным.

Лучше использовать инструменты NebulaAuth.

</details>

<details>

<summary><strong>Можно ли изменить секреты (shared_secret, identity_secret)?</strong></summary>

Нет.

Эти значения создаются Steam при привязке аутентификатора.

Чтобы получить новые, нужно отвязать Steam Guard и привязать его заново.

</details>
